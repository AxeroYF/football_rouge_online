# 场边实验室：开发者工具使用说明

## 打开工具

当前本地地址：

http://127.0.0.1:4310/

如果以后服务已经停止，在项目目录运行：

    npm run devtool

工具只监听 127.0.0.1，并会拒绝非本机请求。它位于独立的 devtool 目录，不会进入正式游戏构建。

## 一、特性卡管理

- 顶部工作区分为“正式卡池”和“开发中”。正式卡仍参与抽取、装备与比赛；开发中草稿完全隔离，不会被游戏或模拟读取。
- “开发中”允许只填写名称、一句效果想法或任意备注；稀有度、类别、位置、羁绊、标签、权重和程序规则均可留空。
- 程序实现草案按原文保存，可以写伪代码、未完成 JSON 或待办说明，不会进行格式校验。
- 草稿提供构思中、待补充、可测试、测试中和待评审五种状态，并显示信息完成度与状态筛选。
- 当前没有自动“并入正式卡池”操作。测试通过后再共同评审字段和程序实现，确认无误后手动合入正式卡。
- 搜索并按稀有度、位置筛选全部特性卡。
- 新建、复制和删除卡牌。
- 调整名称、稳定 ID、稀有度、类别、掉落权重、性质、适用位置、标签和玩家可见说明。
- 直接编辑机器规则 JSON，并即时检查格式。
- 修改稳定 ID 时，所有球员的装备引用会同步更新。
- 删除已装备卡牌时，会明确提示并清理对应球员引用。

## 二、球员卡管理

- 搜索并按球队、位置筛选球员。
- 新建、复制和删除球员卡。
- 调整姓名、位置、球队、首发/替补/预备状态、惯用脚和身高。
- 主位置与副位置统一使用七人制的11个位置：GK、CB/LB/RB、DM/AM/LM/RM、ST/LW/RW。
- 管理体力、状态与受伤倾向。
- 分组调整26项球员能力。
- 每名球员有三个特性槽，候选卡会根据位置自动过滤。
- 球员调整会同步更新对应球队阵容。

## 三、比赛模拟实验室

- 选择主客队与批量模拟场次。
- 选择晴天、雨天、雷暴或雪天预设；当前游戏权重为60% / 15% / 15% / 10%。
- 调整比赛分钟、控球序列、主场优势、场地、降雨、风力、温度和雷击/推进概率。
- 分别调整双方11项战术参数。
- 调整阵型结构、球队状态、教练能力和裁判参数。
- 使用固定随机种子复现同一场比赛。
- 同时输出单场比分、xG、射门、控球、犯规、牌数、事件时间线，以及批量胜平负概率和常见比分。

## 数据与备份

主要数据保存在：

    data/devtool-db.json

每次保存前，上一个版本会复制到：

    data/devtool-db.backup.json

顶部的“导入”和“导出”可以迁移完整开发数据。导入前会进行结构校验。

开发中草稿保存在同一个数据库的 `traitDrafts` 独立集合中；正式游戏只读取 `traitCards`。

## 当前特性模拟范围

v0.2起，开发者后台的单场/批量模拟与玩家侧实时比赛共用同一个七人制比赛引擎。双方都固定使用7名首发、最多4名替补和3个换人名额，并共用26项能力、位置、惯用脚与位置适配算法。

静态属性修改规则已经参与比赛模拟，例如“头球+5”“速度-2”和“提高最高三项非身体能力”。

依赖分钟、比分、射门事件、红牌或一次性重判的临场规则已经保存在卡牌数据中，但尚未全部接入比赛运行时。每次模拟后，结果页会显示本次应用的规则数量和等待接入的规则数量，避免出现隐藏失效。

## 构建与验证

生成静态界面构建：

    npm run build

运行全部自动测试：

    npm test
